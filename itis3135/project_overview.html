Client Project Proposal

Overview

The site is for professionals involved in promoting technology innovation, evaluating emerging technologies, and advising and engaging in the creation of R&D projects and startup ventures. Offering a hub of resources for others developing new technologies and supporting innovation efforts. And, a place where like-minded technologists, product development experts, entrepreneurs, former executives, and prospective investors can explore and create opportunities together.

Site Map

Page Design

Home

  • Purpose: To be an introduction to the brand and navigation hub.
  • Audience/Users: Everyone: potential clients, investors, partners, etc.
  • Content: A landing page featuring a large Logo and subtext as on the current placeholder site atspeedgroup.com. Top-level navigation to each part of the website.
  • Asking For User Data?: No
  • Validation For Data?: N/A
  • UI Elements: It will contain top level menu links to other pages. Some menu items will have drop down menus.
    Nav links: Home, About (Our story, Featured projects), Blog, Login (Members, Partners, Investors), Resources (curated list of links to innovation content).
  • Actions: Navigate to other pages.
  • Special Notes: Needs to look awesome and visually compelling and oozes innovation vibe.

About

  • Purpose: Familiarize visitor with @Speed's activity.
  • Audience/Users: Everyone: potential clients, investors, partners, etc.
  • Content: Description of the @Speed's backstory. Examples of technologies and projects that have been pursued.
  • Asking For User Data?: No
  • Validation For Data?: N/A
  • UI Elements: Standard top-level nav to other parts of the site.
  • Actions: Navigate to other pages.
  • Special Notes: N/A

Blog

  • Purpose: Standard blog functionality. Perhaps a blog service plug-in.
  • Audience/Users: Everyone: potential clients, investors, partners, etc.
  • Content: Posts authored by members.
  • Asking For User Data?: Ideally will include means for members to create or upload a new post.
  • Validation For Data?: Will need to validate if user is a member.
  • UI Elements: Consistent top-level nav to other parts of the site.
  • Actions: Standard top-level nav. Maybe a submit button for new posts.
  • Special Notes: N/A

Login

  • Purpose: ID and PWD restricted access.
  • Audience/Users: Members, Partners, Investors.
  • Content: Login form.
  • Asking For User Data?: Yes - ID and PWD.
  • Validation For Data?: Yes - ID and PWD.
  • UI Elements: Consistent top-level nav to other parts of the site.
  • Actions: Standard top-level nav.
  • Special Notes: N/A

Resources

  • Purpose: Present access to innovation resources.
  • Audience/Users: Everyone: potential clients, investors, partners, etc.
  • Content: Curated links to relevant content sources.
  • Asking For User Data?: No
  • Validation For Data?: N/A
  • UI Elements: Links
  • Actions: Standard top-level nav. Links to relavent content.
  • Special Notes: N/A

Contact

  • Purpose: Provides communication with the site owner.
  • Audience/Users: Everyone: potential clients, investors, partners, etc.
  • Content: Email submittal form.
  • Asking For User Data?: User email.
  • Validation For Data?: Email validation.
  • UI Elements: Submit button.
  • Actions: Standard top-level nav. Submit button sends email to default contact address.
  • Special Notes: N/A

Dynamic Functionality

There will be a blog page that will allow members to post new content. A login page will be required to access this feature. Perhaps a gallery of featured projects on the about page. Lastly, a contact page will allow users to send an email to the site owner.
